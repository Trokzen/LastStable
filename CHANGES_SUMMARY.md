# Резюме изменений: Заполнение таблиц организаций при запуске алгоритма

## Проблема
При запуске нового алгоритма через функцию "Запустить новый алгоритм" в UI заполнялись только таблицы:
- `algorithm_executions` - для записи исполнения алгоритма
- `action_executions` - для записи исполнения действий

Однако НЕ заполнялись новые таблицы:
- `exec_organizations` - для организаций в рамках исполнения
- `exec_organization_files` - для файлов организаций в рамках исполнения

## Решение
Реализовано автоматическое заполнение таблиц `exec_organizations` и `exec_organization_files` при запуске алгоритма.

## Файл который был изменен
- **`db/sqlite_database_manager.py`**

## Детали изменений

### 1. Добавлены 4 новых метода для работы с `exec_organization_files`:

```python
def get_exec_organization_files(self, exec_org_id: int) -> list:
    """Получить все справочные файлы организации в рамках выполнения действия."""
    
def add_file_to_exec_organization(self, exec_org_id: int, file_data: dict) -> int:
    """Добавить справочный файл к организации в рамках выполнения действия."""
    
def get_exec_organization_file_by_id(self, file_id: int) -> dict | None:
    """Получить справочный файл организации выполнения по ID."""
    
def remove_file_from_exec_organization(self, file_id: int) -> bool:
    """Удалить справочный файл из организации выполнения действия."""
```

### 2. Модифицирован метод `start_algorithm_execution`:

После создания каждого `action_execution` добавлена логика:

1. **Получение организаций**: Выбираются все организации, привязанные к исходному действию из таблицы `organizations`
2. **Создание записей организаций**: Для каждой организации создается запись в `exec_organizations` с сохранением всех данных:
   - `name` - наименование организации
   - `phone` - телефон
   - `contact_person` - контактное лицо
   - `notes` - примечания
3. **Создание записей файлов**: Для каждого файла организации (из таблицы `organization_files`) создается соответствующая запись в `exec_organization_files`:
   - `file_path` - путь к файлу
   - `file_type` - тип файла (word, excel, pdf, image, other)

## Логика выполнения

```
START_ALGORITHM_EXECUTION
├── Создание algorithm_execution
├── Для каждого действия (action):
│   ├── Создание action_execution
│   ├── Получение организаций для действия
│   └── Для каждой организации:
│       ├── Создание exec_organizations
│       ├── Получение файлов организации
│       └── Для каждого файла:
│           └── Создание exec_organization_files
└── Завершение транзакции
```

## Пример логирования

При запуске алгоритма с организациями будет выводиться:
```
SQLiteDatabaseManager: Создана организация [название] с ID [ID] для action_execution ID [ID].
SQLiteDatabaseManager: Добавлен файл ID [ID] ([путь]) для организации выполнения ID [ID].
SQLiteDatabaseManager: Обработано [N] организаций(ы) для действия [ID].
```

## Важные замечания

1. **Модель данных**: Таблицы `exec_organizations` и `exec_organization_files` создают снимки (snapshots) данных организаций на момент запуска алгоритма
2. **Целостность данных**: Все операции выполняются в одной транзакции, поэтому либо всё сохраняется, либо ничего (благодаря `with conn:`)
3. **PostgreSQL**: На данный момент изменения сделаны только для SQLite. PostgreSQL менеджер использует другую схему данных и может требовать отдельного обновления
4. **Синтаксис**: Проверено на правильность синтаксиса Python - ошибок не найдено

## Файлы и таблицы

### Задействованные таблицы (шаблоны):
- `algorithms` - шаблон алгоритма
- `actions` - действия алгоритма
- `organizations` - организации для действий
- `organization_files` - файлы организаций

### Задействованные таблицы (исполнения):
- `algorithm_executions` - исполнение алгоритма
- `action_executions` - исполнение действий
- `exec_organizations` - организации в исполнении (НОВОЕ)
- `exec_organization_files` - файлы организаций в исполнении (НОВОЕ)

## Тестирование

Для тестирования:
1. Создайте алгоритм с действиями
2. Добавьте организации к действиям
3. Добавьте файлы к организациям
4. Запустите алгоритм через функцию "Запустить новый алгоритм"
5. Проверьте, что в базе данных создались записи в таблицах `exec_organizations` и `exec_organization_files` с правильными данными
